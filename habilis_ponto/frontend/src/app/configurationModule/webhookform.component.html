Configuração de webhooks e API

Chave para autenticação na fila (api_key)
Hook para alteração no estado da fila
Evento disparado quando há alguma mudança no estado da fila, como por exemplo, a perda de autenticação, desconexão do smartphone ou do serviço de mensageria.
Hook para alteração no estado da fila
Evento disparado quando há alguma mudança no estado da fila, como por exemplo, a perda de autenticação, desconexão do smartphone ou do serviço de mensageria.
Hook para chamada recebida
Evento disparado quando um cliente faz uma chamada de áudio ou de vídeo para o serviço de uma fila.
Hook para cliente digitando
Evento disparado quando um cliente está digitando em um atendimento aberto na fila.
Hook para confirmação de mensagem recebida pelo servidor
Evento disparado quando o serviço de mensageria confirma que seu servidor recebeu uma mensagem enviada. Nesse momento ainda não há confirmação de que a mensagem já foi entregue ao usuário.
Hook para confirmação de mensagem recebida pelo cliente
Evento disparado quando o serviço de mensageria confirma que o cliente recebeu uma mensagem enviada.
Hook para mensagem recebida no sistema
Evento disparado quando uma mensagem é recebida na fila.
Hook para comunicação de mensagem excluída pelo cliente
Evento disparado quando o cliente solicita a exclusão de uma mensagem ao serviço de mensageria.
Hook para confirmação de mensagem lida pelo cliente
Evento disparado quando o serviço de mensageria confirma que o cliente leu uma mensagem enviada.
Hook para a abertura de um novo atendimento
Evento disparado quando um novo atendimento é aberto no sistema.
Hook para fechamento de um atendimento aberto
Evento disparado quando um atendimento da fila é encerrado.
